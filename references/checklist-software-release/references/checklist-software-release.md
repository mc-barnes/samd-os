Summary
This checklist is used to verify that documentation and activities are complete
before releasing a new version of the product.
As with all regulatory documents, it’s more about the content than
about the tool. You don’t have to fill this checklist out every time
in Word / GDocs / etc., but could embed it in your Jira / GitHub
workflow.
The main point is that, at minimum, the items below
should be checked (and documented) before you release a new version
of your software.
Feel free to add further rows in the checklist if they make sense for
your company. This template is pretty much the bare minimum to
be 62304-compliant.
