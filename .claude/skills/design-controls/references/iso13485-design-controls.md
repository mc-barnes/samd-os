# ISO 13485 Design Controls (Clause 7.3)

## Process Overview

```
User Needs → Design Input → Design Output → Verification → Validation
                                                              ↓
                                                         Design Transfer
```

Each transition requires documented review with objective evidence.

## Clause 7.3 Mapping

| Subclause | Activity | Required Records |
|-----------|----------|-----------------|
| 7.3.1 | Design planning | Plan with stages, responsibilities, review points |
| 7.3.2 | Design input | Requirements list, regulatory reqs, risk analysis input |
| 7.3.3 | Design output | Specifications that meet inputs, acceptance criteria |
| 7.3.4 | Design review | Meeting minutes, attendees, action items, decisions |
| 7.3.5 | Design verification | Test protocols, results, pass/fail vs. design inputs |
| 7.3.6 | Design validation | Use testing, clinical evidence, user acceptance |
| 7.3.7 | Design transfer | Manufacturing specs, IFU, labeling |
| 7.3.8 | Design changes | Change request, impact assessment, re-verification scope |
| 7.3.9 | Design history file | Complete record of all above |

## FDA 21 CFR 820.30 Mapping

| FDA Section | ISO 13485 | Key Difference |
|------------|-----------|----------------|
| 820.30(c) Design input | 7.3.2 | FDA emphasizes "incomplete, ambiguous, conflicting" review |
| 820.30(d) Design output | 7.3.3 | FDA requires reference to acceptance criteria |
| 820.30(e) Design review | 7.3.4 | FDA requires independent reviewer |
| 820.30(f) Design verification | 7.3.5 | Essentially identical |
| 820.30(g) Design validation | 7.3.6 | FDA requires "initial production units" |

## Common 483 Observations
1. Incomplete traceability (user need → design input → verification)
2. Design reviews without documented attendee list or action items
3. Validation performed on non-production-equivalent units
4. Missing risk analysis input to design input process
5. Design changes without re-evaluation of affected V&V
