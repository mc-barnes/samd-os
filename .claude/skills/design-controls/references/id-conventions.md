# ID Conventions for Design Controls

## Format
- User Needs: `UN-001`, `UN-002`, ...
- Design Inputs: `DI-001`, `DI-002`, ...
- Design Outputs: `DO-001`, `DO-002`, ...
- Verification: `VER-001`, `VER-002`, ...
- Validation: `VAL-001`, `VAL-002`, ...
- Hazards: `HAZ-001`, `HAZ-002`, ...
- Risk Controls: `RC-001`, `RC-002`, ...
- Change Requests: `CR-001`, `CR-002`, ...

## Linking Rules
- Each DI links to one or more UN (many-to-one or many-to-many)
- Each DO links to exactly one DI (one-to-one)
- Each VER links to exactly one DO (one-to-one)
- Each VAL links to one or more UN (validates the original need)

## Numbering
- Sequential within each type
- Never reuse IDs (even if item is deprecated/removed)
- Use prefix for traceability in formulas: `=VLOOKUP("DI-001", Design_Inputs!A:D, 2, FALSE)`
