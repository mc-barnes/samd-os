# Design Review Checklists

## PDR Checklist (25 items)
1. User needs document complete
2. Design inputs derived from user needs
3. Intended use statement defined
4. User population characterized
5. Use environment described
6. Regulatory pathway identified
7. Applicable standards listed
8. Risk management plan created
9. Initial hazard analysis performed
10. SW safety classification determined
11. System architecture defined
12. Key algorithms identified
13. Data flow documented
14. Interface specifications drafted
15. Performance requirements specified
16. Security requirements identified
17. Privacy/HIPAA assessment done
18. V&V strategy outlined
19. Clinical evidence plan (if needed)
20. Development timeline realistic
21. Resource plan adequate
22. Previous review actions closed
23. Design history file initiated
24. Configuration management set up
25. Supplier/third-party assessment (if applicable)

## CDR Checklist (31 items)
All PDR items (verified/updated) PLUS:
26. Detailed design specifications complete
27. All design inputs have corresponding outputs
28. Traceability matrix complete (UN->DI->DO->VER->VAL)
29. Design inputs are unambiguous, complete, non-conflicting, and have measurable acceptance criteria (per FDA 820.30(c))
30. Risk analysis updated with design details
31. All unacceptable risks have control measures
32. Risk controls traced to requirements
33. V&V protocols written and reviewed
34. Test environment specified
35. Acceptance criteria quantified
36. Algorithm performance targets set
37. Data quality requirements defined
38. Integration test plan complete
39. Regression test strategy defined
40. Cybersecurity analysis complete
41. Labeling/IFU drafted
42. Manufacturing/deployment plan drafted
43. Training materials outlined
44. Post-market surveillance plan initiated
45. All CDR action items from PDR closed
46. Independent review of critical safety items
47. Biocompatibility assessment (if applicable)
48. Electrical safety assessment (if applicable)
49. EMC assessment (if applicable)
50. Software of Unknown Provenance (SOUP) list complete
51. SBOM (Software Bill of Materials) generated
52. Usability engineering file initiated
53. Clinical evaluation report drafted
54. Regulatory submission timeline set
55. Budget and resource reconfirmed
56. Stakeholder sign-off on design direction

## FDR Checklist (36 items)
All CDR items (verified/updated) PLUS:
57. All verification tests executed and passed
58. All validation tests executed and passed
59. Verification vs. validation clearly distinguished — verification confirms design outputs meet design inputs (built right); validation confirms device meets user needs (built the right thing). Evidence shows distinct test protocols for each.
60. Traceability matrix fully populated (no gaps)
61. Risk management report complete
62. Overall residual risk acceptable
63. Design transfer documentation complete
64. Labeling/IFU finalized
65. Training materials finalized
66. Post-market surveillance plan finalized
67. Regulatory submission package assembled
68. Clinical evidence sufficient
69. Independent review complete
70. All open action items from CDR closed
71. No unresolved non-conformances
72. Production/deployment environment validated
73. Backup and recovery procedures tested
74. Cybersecurity measures verified
75. Data retention/destruction plan
76. Customer support plan ready
77. Field safety corrective action plan
78. All SOUP/SBOM items assessed
79. Usability validation complete
80. Human factors summative evaluation complete
81. Software release notes prepared
82. Design history file complete
83. Quality system audit findings resolved
84. Regulatory authority correspondence reviewed
85. Market authorization application ready
86. Go-to-market plan finalized
87. Launch readiness confirmed by all stakeholders
88. Post-launch monitoring plan active
89. Complaint handling process ready
90. Adverse event reporting process ready
91. Product recall procedure documented
92. Executive sign-off obtained
